## Coverage note

ACS automated checks cover a subset of the full framework — those controls for which evidence can be reliably detected from code, configuration, and documentation files in the repository. Controls that require operator-side artefacts (Business Associate Agreements, signed risk-management policies, physical-access logs, training records) are **out of automated scope** and must be evidenced through the customer's own GRC programme. For the full control set see the ACS aggregate output at `acs-scan-2026-05-08-postfix/framework__eu_ai_act.json`.

FAILs in this report represent **automated check misses** against code or documentation files in this repository. Many are recoverable by adding the expected file (e.g. `docs/asset_inventory.md`) or by adjusting a single code-search pattern; some indicate genuine gaps tracked in the v2.24.0 backlog. Per `feedback_no_fake_compliance_docs.md` they are reported as FAIL honestly, not silently downgraded.
