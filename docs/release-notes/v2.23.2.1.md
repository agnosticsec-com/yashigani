## Summary

v2.23.2.1 is a single-fix Helm chart patch. It corrects image digest pointers in `helm/yashigani/values.yaml` that were left pointing at v2.23.1 digests after the v2.23.2 GA tag was cut. The project maintainer accepted option (ii): cut a v2.23.2.1 patch tag, keeping v2.23.2 immutable.

**Delivered by:** PR #77 (`chore/v232-helm-digest-pin-fast-follow`)

---

## Fix

### (#77) `helm/yashigani/values.yaml` — v2.23.1 digest references corrected

**Root cause:** The v2.23.2 GA chart at tag `v2.23.2` referenced v2.23.1 image digests for both `yashigani-gateway` and `yashigani-backoffice`. Kubernetes Helm deployments therefore pulled the v2.23.1 container images when running the v2.23.2 chart — a chart/binary version mismatch invisible to operators without explicit image inspection.

**Impact:** Helm-only. `docker-compose.release.yml` image pins were correct at GA and are unchanged by this patch.

**Corrected digests pinned by this patch:**

| Image | Tag | Correct v2.23.2 Digest |
|-------|-----|------------------------|
| `yashigani-gateway` | `2.23.2` | `sha256:7ffe5b92b23224a5fdcf86bf2570345d979ca2f9f06beda52447efc6ef5b688c` |
| `yashigani-backoffice` | `2.23.2` | `sha256:34f4e68dac35f6e045bfe6277cd9725340e0e662361cdfb81b2057d3a0ebd0dc` |

No other files are changed. Compose, Helm subchart values, documentation, and all application code are identical to v2.23.2.

## Upgrade Path

### Helm

```sh
helm repo update
helm upgrade yashigani helm/yashigani/ \
  --version 2.23.2.1 \
  -f helm/yashigani/values.release.yaml
```

No image rebuild required. No config changes required. No database migrations.

Verify the deployed digest after upgrade:

```sh
kubectl get pod -l app=yashigani-gateway \
  -o jsonpath='{.items[0].status.containerStatuses[0].imageID}'
# Expected suffix: sha256:7ffe5b92b23224a5fdcf86bf2570345d979ca2f9f06beda52447efc6ef5b688c
```

### Docker / Podman Compose

No action required. Image pins in `docker-compose.release.yml` were already pointing at v2.23.2 digests. This patch does not change them.
