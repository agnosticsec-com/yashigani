## Highlights

### Admin-triggered secret rotation (`feat(secrets): rotate-secrets`)

Yashigani v2.23.3 introduces zero-downtime, admin-triggered rotation of
infrastructure credentials via the backoffice API. No nuke-and-rebuild required.

**Supported secrets:**

| Secret | Mechanism | Zero-downtime? |
|---|---|---|
| `postgres_password` | `ALTER USER` via direct Postgres connection; pgbouncer restart | Yes (dual-credential window) |
| `redis_password` | `CONFIG SET requirepass` in-flight | Yes |
| `jwt_signing_key` | File hot-swap; existing tokens valid until expiry | Yes |
| `hmac_key` | File hot-swap; brief ~1-2 request window on reload | Near-zero |
| `all` | Sequential rotation of all four | Inherits per-secret |

## Security

### Secret rotation API (ASVS V6.8.4, V7.1.2, V12.4.1)

**Finding addressed:** Secrets were only rotatable via full nuke-and-reinstall
(`install.sh`). This created operational pressure to avoid rotation, increasing
exposure windows.

**Change:** New endpoint `POST /api/v1/admin/secrets/rotate` with:
- Step-up TOTP required (ASVS V6.8.4 re-authentication before critical operations).
- Fail-closed: if rotation fails mid-way, the old secret is reverted. If revert
  also fails, the event is logged `CRITICAL` and the admin is alerted.
- Atomic secret file writes: temp file + `rename(2)` — concurrent readers never
  see a partial secret.
- Immutable audit trail: `SECRET_ROTATION_REQUESTED` → `SECRET_ROTATION_SUCCEEDED`
  or `SECRET_ROTATION_FAILED` in the SHA-384 audit chain. Secret values are never
  stored in audit events.
- Body limit: 256 bytes (ASVS 4.3.1).

**Password charset:** New passwords use `A-Za-z0-9!*,-._~` per
`feedback_password_charset.md` — URL-safe (no `%` encoding in DSNs), shell-safe
(no `$`, backtick, `|`, `&`, `;` in double-quoted strings), 48 characters.

**CLI wrapper:** `scripts/rotate-secret.sh <secret-name>` wraps the API and
runs `docker/podman compose restart` for the affected services from the host.

**Documentation:** See `docs/yashigani_install_config.md §28` for full
operational runbook including failure recovery for the `revert_failed=true` case.
