## Methodology

Each control was verified using automated static analysis against the Yashigani codebase:

- **file_contains** — Read a specific file and search for a regex pattern
- **any_file_contains** — Search a directory tree for a regex pattern
- **file_exists** — Check if a file or glob pattern matches
- **none_file_contains** — Verify a pattern is absent (negative check)
- **all_of** — All sub-checks must pass
- **any_of** — At least one sub-check must pass
- **always_pass** — Control is not applicable (marked N/A with reason)
- **always_fail** — Known gap (marked as FAIL with explanation)
